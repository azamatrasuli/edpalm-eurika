## Часть 1. Законодательная база

### 1.1. ФЗ-152 "О персональных данных"

**Что считается ПДн:** любая информация, по которой можно идентифицировать человека. ФИО + телефон, Telegram ID + имя, сообщения чата привязанные к профилю — всё это ПДн.

**Основания для обработки (ст. 6):**
- Согласие субъекта (п. 1) — для новых лидов через чат-бот
- Исполнение договора (п. 5) — для действующих учеников
- Законный интерес (п. 7) — для аналитики (с осторожностью)

**Согласие (ст. 9, изменения с 01.09.2025):**
- Отдельный документ (не внутри оферты)
- Конкретное, информированное, сознательное
- Чекбокс пустой по умолчанию
- Логирование факта и времени получения
- Каждая цель обработки — отдельное согласие

**Локализация (ч. 5 ст. 18, усилена с 01.07.2025):**
- Первичный сбор ПДн граждан РФ — ТОЛЬКО на серверах в РФ
- Формулировка: "не допускается" использование БД вне РФ
- Штраф: до 6 млн (первое), до 18 млн (повторное)

**Трансграничная передача (ст. 12):**
- Уведомление РКН до передачи
- США НЕ в списке "адекватных" стран
- Австралия — в списке (Категория 2)
- Для стран вне списка — одобрение РКН (10 дней, может запретить)

**Утечки (ст. 21):**
- 24 часа — первичное уведомление РКН
- 72 часа — результаты расследования
- Штраф за неуведомление: 1–3 млн руб.

**Штрафы за утечки (ФЗ-420 от 30.11.2024, с 30.05.2025):**
- 1К–10К субъектов: 3–5 млн руб.
- 10К–100К: 5–10 млн руб.
- 100К+: 10–15 млн руб.
- Повторная утечка: 1–3% выручки (мин. 20 млн, макс. 500 млн руб.)

**Уголовная ответственность (ст. 272.1 УК, с 11.12.2024):**
- Незаконный оборот ПДн: до 4 лет лишения свободы
- Данные несовершеннолетних: до 5 лет
- С трансграничной передачей: до 8 лет
- Организованная группа / тяжкие последствия: до 10 лет

**Автоматизированные решения (ст. 16):**
- Решения на основе автоматизированной обработки — только с письменного согласия
- Право субъекта на информацию о логике и на обжалование

**Права субъектов:**
- Доступ к данным, уточнение, удаление (30 дней)
- Отзыв согласия в любой момент
- Запрет маркетинга
- Запрет автоматизированных решений

### 1.2. Иностранные сервисы

**OpenAI:** Россия заблокирована, доступ через VPN нарушает ToS. Передача ПДн в США — незаконная трансграничная передача.

**Supabase (AWS Сидней):** Прямое нарушение локализации — первичная БД вне РФ.

**Render.com / Vercel:** Серверы в США. Обработка и хостинг ПДн вне РФ.

**amoCRM:** Российская компания, серверы в РФ — ОК.

### 1.3. Реклама (38-ФЗ)

**Follow-up сообщения** (проактивные, без запроса клиента) = реклама:
- Маркировка "Реклама" + erid-токен
- Отдельное согласие на рекламу
- Штраф: до 500 тыс. руб.

**Ответы на прямые вопросы** клиента — НЕ реклама.

### 1.4. WhatsApp

С сентября 2025 Meta (WhatsApp) — "нежелательная" организация. Коммерческие рассылки запрещены. Убрать из V2 плана.

### 1.5. Закон об ИИ (проект, ~01.09.2027)

- Обязательная маркировка AI-контента
- Уведомление пользователя что общается с ИИ
- Классификация рисков ИИ-систем

### 1.6. Защита потребителей

- Клиент может расторгнуть договор в любой момент
- Возврат полной суммы минус фактические расходы
- Бот не может отказать в возврате

### 1.7. Регистрация в РКН

Обязательна для всех операторов ПДн с 30.05.2025. Подача через pd.rkn.gov.ru с УКЭП.

## Часть 2. Аудит потоков ПДн в Eurika

### 2.1. Что отправляется в OpenAI (КРИТИЧНО)

На КАЖДЫЙ запрос к GPT-4o отправляются:

**В системном промпте:**
- display_name (имя клиента)
- phone (номер телефона)
- actor_id (telegram:123456 или portal:user123)
- channel (telegram/portal/external)

**В CRM-контексте:**
- contact_name, contact_id из amoCRM
- deal.name, deal.product, deal.amount

**В профильном контексте:**
- client_type, user_role
- phone (нормализованный), FIO
- children: [{fio, grade}] для каждого ребёнка
- dms_verified, verification_status, product_name

**В памяти:**
- Факты из memory_atoms: "Иванов хочет Экстернат 5 класс"
- Саммари прошлых разговоров с именами и деталями

**В истории:**
- До 50 сообщений полного текста (user + assistant)
- Tool call результаты (содержат имена, телефоны, данные учеников)

**В tool calls:**
- Аргументы: phone, telegram_id, contact_id
- Результаты: полные профили (ФИО, телефоны, классы, школы, Moodle ID)

**Голос (Whisper):**
- Аудио-файл целиком отправляется в OpenAI
- Если клиент произносит имя/телефон — данные захвачены

**TTS:**
- Текст ответа агента отправляется в OpenAI
- Если ответ содержит имена/данные — они в запросе

**Embeddings:**
- Текст сообщения пользователя → OpenAI embeddings API
- Тексты саммари и фактов → OpenAI embeddings API

### 2.2. Что хранится в Supabase (текущая БД вне РФ)

|---------|-----|-------|
| chat_messages | Полный текст сообщений с именами, телефонами | 4,784 |
| agent_user_profiles | ФИО, телефон, дети, классы, DMS-данные | 59 |
| agent_memory_atoms | Факты: "Иванов → ученик → 5 класс" | 582 |
| agent_conversation_summaries | Саммари с контекстом и именами | 203 |
| agent_contact_mapping | actor_id → contact_name, contact_id | 561 |
| agent_chat_mapping | actor_id → contact_name, lead_id | 939 |
| conversations | actor_id, title (из сообщения), running_summary | 1,376 |
| agent_payment_orders | actor_id, product, amount, dms_contact_id | 9 |

### 2.3. Что в логах

- Телефоны (onboarding.py: `phone=%s`)
- Аргументы инструментов с phone/contact_id (tools.py: `args=%s`)
- Имена контактов (dms.py: `found contact_id=%d (%s %s)`)
- Первые 60 символов сообщений (imbox.py)

### 2.4. RAG — чисто

Knowledge chunks содержат ТОЛЬКО контент базы знаний (продукты, цены, FAQ). ПДн отсутствуют. Namespace-изоляция работает (sales/support/teacher/shared).
