## Правила агента (из prompt.py)

- **Квалификация:** класс → цель → формат → имя → телефон (естественно, не анкета)
- **ВСЕГДА** вызывать `search_knowledge_base` на продуктовые вопросы
- **Не придумывать** цены, скидки, имена учителей
- **Эскалация:** негатив, "хочу человека", нет ответа в KB, скидка не из KB
- **Тон:** родители — "вы", подростки 9-11 — "ты"

## Как создать тестовый раннер

Создай `test_runner.py` в backend/:
1. Генерация external-токенов (HMAC-SHA256)
2. `POST /api/v1/conversations/start` → conversation_id + greeting
3. `POST /api/v1/chat/stream` → SSE парсинг (event: type\ndata: json)
4. Поддержка multi-turn (несколько сообщений в одном диалоге)
5. Возврат JSON: scenario_id, responses[], tools_called[], errors[], escalations[], payment_cards[], suggestions[]

## Критерии качества

### По инструментам:
- `search_knowledge_base` ОБЯЗАТЕЛЬНО вызывается на КАЖДЫЙ вопрос о продуктах/ценах/оплате/аттестации
- `create_amocrm_lead` вызывается когда клиент даёт имя + телефон
- `escalate_to_manager` на скидки, негатив, "хочу человека", нет в KB

### По контенту:
- ВСЕ цены совпадают с KB (0 галлюцинаций)
- Не придумывает скидки, промокоды, имена учителей
- Не ругает конкурентов
- Off-topic отклоняется мягко

### По тону:
- Родители: "вы", тепло, экспертно
- Подростки 9-11: "ты", дружелюбно
- Эмоциональные: сначала эмпатия, потом решение
