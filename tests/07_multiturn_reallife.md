## Критерии

- Каждый диалог = РЕАЛЬНЫЙ клиент (не тестовый запрос)
- Контекст сохраняется от сообщения к сообщению
- Тон адаптируется под клиента
- Квалификация нативная, не допрос
- Нет противоречий в ответах в рамках одного диалога
- create_amocrm_lead вызывается при имени+телефоне
- escalate_to_manager при негативе/запросе человека

## Инструкции

1. Создай test_runner.py с multi-turn (4-8 сообщений)
2. По 5 субагентов, 5с пауза
3. Каждый субагент = 2-3 полных диалога
4. Проверяй coherence внутри диалога (нет противоречий)
5. Баг → фикс → retest
6. Удали test_runner.py
