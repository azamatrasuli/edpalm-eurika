## Порядок квалификации (из prompt.py)

Агент должен вплетать квалификационные вопросы в ПОЛЕЗНЫЙ разговор. Порядок:
1. **КЛАСС** → "В каком классе ребёнок?" — сразу расскажи что есть для этого класса
2. **ЦЕЛЬ** → "Основная программа или подготовка к экзаменам?" — предложи формат
3. **ФОРМАТ** → "Нужны живые уроки или достаточно самостоятельного?" — сузь до тарифа
4. **ИМЯ** → "Как зовут ребёнка? Подготовлю предложение" — персонализация
5. **ТЕЛЕФОН** → "На какой номер оформить?" — ТОЛЬКО при готовности к действию

**Правила:**
- Максимум 1 квалификационный вопрос на сообщение
- СНАЧАЛА ответь на вопрос клиента, ПОТОМ задай свой
- Пропускай то, что уже известно из профиля
- НЕ спрашивай: адрес, страну, количество детей (если клиент не упомянул)
- Телефон — ПОСЛЕДНИЙ (только при готовности к оплате/заявке)

## Как создать тестовый раннер

Создай `test_runner.py` в backend/:

2. **Multi-turn обязательно**: каждый сценарий = 3-7 сообщений в одном диалоге

## Методология проверки

Для КАЖДОГО multi-turn сценария:
1. Проверь что квалификация идёт в правильном порядке (класс → цель → формат → имя → телефон)
2. Проверь что агент НЕ задаёт лишних вопросов
3. Проверь что агент отвечает на вопрос клиента ПЕРЕД квалификацией
4. Проверь что тариф подобран ПРАВИЛЬНО для данных клиента
5. Проверь search_knowledge_base вызывается на продуктовые вопросы
6. Проверь тон: "вы" для родителей, "ты" для подростков

## Критерии качества

- Квалификация нативная — как живой разговор, не анкета
- Тариф подобран ТОЧНО (Выпускник для 9/11, Заочный для Москвы, и т.д.)
- НЕТ лишних вопросов (если класс уже назван — не переспрашивать)
- Телефон просится ТОЛЬКО при готовности к действию
- Эмпатия при проблемах (буллинг, СДВГ, финансы)
- Цены из KB, не придуманные
- create_amocrm_lead при имени+телефоне
