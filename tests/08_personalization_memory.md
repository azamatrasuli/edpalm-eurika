## Механизмы персонализации

### 1. Greeting (chat.py)
- Время: утро (6-12) / день (12-18) / вечер (18-6) — Moscow UTC+3
- Канал: Telegram (неформальный) / Portal (с именем) / External (generic)
- Профиль: если DMS verified → "Вижу, {ребёнок} в {класс} классе"
- Роль: sales vs support — разные приветствия

### 2. Identity context (llm.py)
- channel, display_name, phone → системное сообщение LLM

### 3. Profile context (onboarding)
- client_type, user_role, phone, FIO, grade, children[], dms_verified

### 4. Memory (memory.py)
- **Summaries:** vector search по прошлым диалогам (top_k=3, threshold=0.4)
- **Atoms:** факты (subject-predicate-object) с recency decay (half-life 30 дней)
- **Scoring:** similarity×0.6 + recency×0.25 + type_boost×0.15
- **Cache:** TTL 5 мин по actor_id:role:query_hash

### 5. CRM context (chat.py)
- contact_id, contact_name, active_deal (product, amount, status)

## Как создать тестовый раннер

test_runner.py: multi-turn, external tokens + portal JWT, **один actor_id для серии тестов** (чтобы проверить memory между диалогами).
