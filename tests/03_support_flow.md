## Как создать тестовый раннер

Создай `test_runner.py` в backend/:
1. Генерация external-токенов (HMAC-SHA256 с `external_link_secret`)
2. Вызов API: `/api/v1/conversations/start` (role=support), `/api/v1/chat/stream`
3. SSE парсинг: `event: type\ndata: json`
4. Multi-turn поддержка
5. JSON результат: tools_called, responses, escalations, errors

## Критерии качества

### По инструментам:
- `search_knowledge_base` на ВСЕ вопросы покрытые KB (регистрация, документы, аттестация, СО/ЗО)
- `get_client_profile` когда клиент даёт телефон
- `create_amocrm_ticket` для действий требующих человека (справки, ошибки, смена тарифа)
- `escalate_to_manager` на негатив, "хочу человека", возвраты, юридические угрозы

### По контенту:
- Ответы совпадают с KB (сроки, адреса, процедуры)
- Не придумывает процедуры которых нет
- Честно говорит "не знаю, подключаю специалиста" если нет в KB

### По тону:
- Всегда эмпатия перед решением при негативе
- Support: спросить телефон рано (1-2 сообщение) для идентификации
- Спокойно, фактически, без канцелярита

## Инструкции по работе

1. Изучи: `support_staff/knowledge_base/01_support_faq.md`, `support_staff/TZ.md`, `backend/app/agent/prompt.py` (SUPPORT_ROLE_PROMPT)
2. Создай test_runner.py (role=support)
3. Запускай по 5 субагентов, 5с пауза между сценариями
4. Каждый ответ сверяй с KB — цитаты должны совпадать
5. Баг → фикс → retest
6. Итог: PASS/FAIL + баги
7. Удали test_runner.py
